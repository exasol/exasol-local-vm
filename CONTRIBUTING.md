# Contributing to Exasol Local VM

Thank you for your interest in contributing. We welcome bug reports, fixes,
documentation improvements, and well-scoped feature proposals.

This repository is still being prepared for public development. Some build,
packaging, and runtime details may change while the project is cleaned up.
Please keep contributions focused and coordinate larger changes through an
issue first.

## How to Contribute

1. Fork the repository and clone your fork.
2. Create a branch for your changes: `git checkout -b fix/short-description`.
3. Make focused changes with clear commit messages.
4. Run the relevant local checks for the area you changed.
5. Submit a pull request using the pull request template.

## Pull Request Guidelines

- Keep pull requests focused on one change.
- Explain what changed and why.
- Add or update tests when changing behavior.
- Update documentation when user-facing behavior changes.
- Reference related issues using `#issue-number`.
- Use clear commit messages. Conventional Commits are preferred, for example
  `feat:`, `fix:`, `docs:`, `test:`, `refactor:`, or `chore:`.

## Code Review Process

- All submissions require review before merging.
- Changes to CI/CD, release, build, signing, security, and other sensitive
  areas may require approval from the repository code owners.
- Please respond to review feedback with either an update or a short
  explanation of the tradeoff.

## Reporting Issues

- Use GitHub Issues for bugs, feature requests, and questions.
- Search existing issues first to avoid duplicates.
- For bugs, include reproduction steps, host operating system, architecture,
  runtime environment, and relevant logs.
- For feature requests, describe the use case and expected behavior.

## Community Guidelines

Be respectful, professional, and constructive. Assume good intent, keep
technical discussions focused on evidence, and help maintain a useful project
environment for contributors and users.
